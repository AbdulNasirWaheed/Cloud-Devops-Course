# Day 27: Real-Time Log Analysis & Text Processing Using sed in Linux

**#100DaysOfDevOps | DevOps & Cloud Journey**

## 📌 Objective
Master the **sed** command for powerful stream editing, text transformation, configuration management, and real-time log processing.

## 🏭 Industry Scenario
As a Junior DevOps Engineer at a cloud-based SaaS company, I used sed to:
- Update configuration files efficiently
- Sanitize sensitive information in logs
- Modify deployment scripts
- Filter and analyze production logs

## ✅ What I Practiced

### Part 1 — sed Basics
- Printing, deleting, and basic substitution
- Global replacements (`/g`)

### Part 2 — Intermediate & Advanced Usage
- In-place editing (`-i`)
- Inserting lines before/after matches
- Multiple commands, regex, and piping
- Commenting lines and appending content

### Part 3 — Real Cloud Admin Tasks
- Updated `nginx.conf` (IP and port changes)
- Sanitized passwords in `.env` and `app.log`
- Modified `deploy.sh` (environment, port, logging)
- Updated Terraform security rules
- Filtered error logs from access logs

## 🗂️ Repository Contents
- `users.txt`, `webservers.txt`, `nginx.conf`

- `.env`, `security_rules.tf`, `config.txt`
- Practice commands and notes
