🐧 Day 16 of My DevOps Journey — Linux File System, User & Group Management on a Real Ubuntu Server! ✅

Today I stepped into the role of a Junior DevOps Engineer onboarding a new Ubuntu 24.04 server for a cloud startup — and managed real user & group permissions from scratch! 🚀

📋 What I practiced today:
✅ Explored the Linux File System Hierarchy (/, /home, /etc, /var, /usr, /bin)
✅ Navigated using both Absolute & Relative Paths
✅ Created a new user: john_dev (sudo adduser)
✅ Set & verified user password (sudo passwd)
✅ Verified user in /etc/passwd (grep, cat)
✅ Created a custom group: devops_team (sudo groupadd)
✅ Verified group in /etc/group (getent, grep)
✅ Added user to sudo group (admin privileges)
✅ Added user to devops_team group (sudo usermod -aG)
✅ Verified all group assignments (groups john_dev)

💡 Key Takeaway:
In real enterprise environments, User & Group Management is not just a Linux skill — it's a Security & Access Control responsibility. Every DevOps/Cloud Engineer must know how to:
→ Control who can run privileged commands
→ Organize team access to shared resources
→ Onboard developers securely on production servers
