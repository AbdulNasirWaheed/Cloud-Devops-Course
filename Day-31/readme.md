Day 31 — Bash Scripting Basics for Automation
Objective  
Provide hands-on experience with shell scripting used by cloud administrators to automate routine tasks in real-world environments (e.g., AWS EC2 Linux servers). By the end of this lesson you will be able to:

Understand shell fundamentals and shell scripting.

Write and execute basic Bash scripts.

Automate user management and backup operations.

Apply scripting skills in cloud environments such as EC2.

Industry Scenario
You are a Cloud Administrator at a mid-sized tech company running multiple Linux-based EC2 instances. Your manager asks you to:

Automate onboarding for new developers (user account creation).

Set up daily backup automation for project files.

Ensure scripts follow security best practices.

What I Learned (Summary)
What a shell is: A command-line interface that interprets commands and runs scripts. Bash is the default shell on most Linux distributions and common in cloud environments.

What shell scripting is: A file of shell commands (usually .sh) that automates repetitive tasks like user management, backups, log rotation, and cloud resource management.

Why it matters: Scripting reduces manual errors, improves reliability, and scales operations across many servers.

Key Concepts Covered
Shebang (#!/bin/bash)

Variables and substitution

Comments for maintainability

User input (read)

Basic arithmetic and command substitution

Secure scripting practices (permissions, avoiding hard-coded secrets)
